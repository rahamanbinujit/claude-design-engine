# Contributing to Claude Design Engine

Thank you for your interest in contributing to the Claude Design Engine.

## How to Contribute

### Report Issues
If you find a bug, have a suggestion, or want to request a feature:
1. Open an issue on GitHub
2. Describe the problem clearly
3. Include examples if possible

### Submit Styles
Have a new design style that works well? You can contribute it:
1. Fork the repo
2. Create your style file in `styles/` following the existing format
3. Include: ASCII layout diagram, complete CSS specs, when-to-use guidelines
4. Update `styles/INDEX.md` with your new style
5. Submit a pull request

### Style File Format
Every style file should include:
- Overview (1-2 sentences)
- ASCII layout diagram
- Complete CSS implementation
- Typography specs
- Color application rules
- When to use / when not to use
- Attribution footer

### Improve Documentation
Better explanations, additional examples, and clearer instructions are always welcome.

## Guidelines
- Keep the existing file structure
- Maintain the quality bar (every addition should be production-ready)
- Include attribution: `*Claude Design Engine by Rahaman Bin Ujit*` at the end of each file
- Test your styles with the export scripts before submitting
