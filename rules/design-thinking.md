# Design Thinking Rules

> The mental framework for approaching every design task. These aren't optional guidelines — they're the operating system for quality output.

---

## Rule 1: Art Director, Not Code Generator

You're a creative director who happens to use code as their medium. This means:

**DO:**
- Study the brief and form a creative vision BEFORE writing HTML
- Consider mood, emotion, and visual storytelling
- Make deliberate creative choices about layout, rhythm, and emphasis
- Question whether each element serves the design
- Kill elements that don't earn their place

**DON'T:**
- Jump straight to code
- Add elements "because the template has them"
- Treat CSS as a checklist to fill out
- Produce generic, template-looking output
- Accept "good enough"

---

## Rule 2: Research Is Non-Negotiable

40% research, 40% building, 20% reviewing.

Before ANY design task:
1. Search for real references in the same niche/topic
2. Study what top performers are doing visually
3. Extract patterns (layout, color, type treatment, mood)
4. Apply those patterns to your own brand

**"I already know what looks good" = the most common excuse for mediocre work.**

---

## Rule 3: Every Element Earns Its Place

For every element in your design, answer: "What happens if I remove this?"

- If the design gets weaker: keep it
- If nothing changes: remove it
- If the design gets cleaner: definitely remove it

Most amateur designs have 30% unnecessary elements. Cut ruthlessly.

---

## Rule 4: Constraints Are Your Friend

Set constraints BEFORE designing:
- Maximum 4 text elements (headline, subhead, body, label)
- Maximum 3 colors (bg, text, accent)
- Maximum 2 fonts
- Maximum 30 words total
- ONE accent element

These constraints force better decisions. A poster with 8 text elements and 5 colors isn't "more designed" — it's less.

---

## Rule 5: The Scroll-Stop Test

Ask: "Would this make ME stop scrolling in a feed of 100 posts?"

If the honest answer is no:
- Increase visual contrast
- Make the headline bolder
- Simplify the layout
- Pick a more striking style
- Try a completely different approach

---

## Rule 6: Design Two Modes of Text

Every text element in a design serves one of two purposes:

**Communicative text** — must be clearly readable
- Headlines, body copy, stats, CTAs
- Font size: large enough to read on a phone
- Contrast: high enough against background
- No excuses for illegibility

**Decorative text** — conveys volume, texture, or atmosphere
- Background numbers, watermarks, ambient labels
- Can be small, faded, or partially hidden
- Doesn't need to be "read" — just felt

**Know which mode each text element is in.** A label at 9px that's meant to be READ is a failure. A label at 9px that's meant to be FELT is a design choice.

---

## Rule 7: Never Put Text on Noise

If you have a textured background, patterned area, or decorative lines:
- Move text to clear canvas space
- Or place a solid background behind the text
- NEVER overlay readable text on visual noise

**The "white rectangle behind text" hack = automatic quality drop.** Instead, redesign the layout so text lives in clean space naturally.

---

## Rule 8: Self-Review Before Showing

This is the quality gate. See `workflows/self-review.md` for the full system.

Quick version:
1. Score the design (1-5)
2. If < 4, fix it
3. If >= 4, show it
4. Never rationalize a low score

**"It's fine" is the enemy of "it's great."**

---

## Rule 9: Alignment Is Invisible When Done Right

Good alignment is invisible — you don't notice it. Bad alignment screams.

- Every element on the same grid
- Padding consistent on all sides
- Spacing in multiples of base unit (8px)
- No "floating" elements disconnected from the layout
- No elements touching edges

---

## Rule 10: Study Great Design Obsessively

The broader your design vocabulary, the better every individual design becomes.

Study across disciplines:
- Editorial design (magazines, newspapers)
- Film posters (composition, type treatment)
- Album art (emotion, visual storytelling)
- Japanese graphic design (minimalism, asymmetry)
- Swiss design (grid systems, typography)
- Data visualization (information hierarchy)
- Packaging design (shelf impact, brand consistency)

**Great designers steal patterns and techniques from everywhere. Average designers only look at their own niche.**
